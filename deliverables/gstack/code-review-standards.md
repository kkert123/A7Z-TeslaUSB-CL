# A7Z TeslaUSB 代码审查标准与流程 v1.0

> 适用于 TeslaUSB 项目的系统化代码审查机制

## 一、审查等级

| 等级 | 触发条件 | 审查者 | 深度 |
|------|---------|--------|------|
| **L1 快速审查** | 单文件 <50行修改、热修复 | 自动化 | 语法+安全扫描 |
| **L2 标准审查** | 单模块修改、新增路由 | 同行 | L1 + 逻辑+边界 |
| **L3 深度审查** | 多模块联动、架构变更、上线前 | 专家团队 | L2 + 架构+性能+安全审计 |
| **L4 上线全检** | 版本发布前 | 3专家并行 | L3 + OWASP+QA+代码健康度 |

## 二、审查清单 (Checklist)

### 2.1 安全 (Security) — P0
- [ ] 无硬编码凭据 (密钥/密码/token)
- [ ] 输入验证 (路径遍历、SQL/Shell注入)
- [ ] 认证/授权覆盖所有敏感端点
- [ ] CSRF 保护 (POST 端点)
- [ ] 敏感数据不通过 API/login 泄露
- [ ] 密码使用哈希存储 (werkzeug.security)

### 2.2 正确性 (Correctness) — P0
- [ ] 导入链完整 (无 NameError/ImportError)
- [ ] 变量名一致 (参数与使用处对齐)
- [ ] Jinja2 模板变量与后端 dict 字段对齐
- [ ] 异常处理充分 (无裸 except:pass)
- [ ] 边界条件 (空值/零/None/空列表)

### 2.3 可靠性 (Reliability) — P1
- [ ] 资源释放 (文件句柄/subprocess/线程)
- [ ] 进程泄漏防护 (GeneratorExit/atexit)
- [ ] 日志轮转 (RotatingFileHandler)
- [ ] 优雅降级 (外部服务不可用时的 fallback)

### 2.4 性能 (Performance) — P1
- [ ] 无同步阻塞 (sleep in request handler)
- [ ] 大目录遍历有缓存
- [ ] 数据库/文件查询无 N+1 问题

### 2.5 可维护性 (Maintainability) — P2
- [ ] 无重复代码 (DRY)
- [ ] 命名清晰一致
- [ ] 模块职责单一
- [ ] 配置与代码分离
- [ ] 无遗留废弃代码/注释

## 三、审查流程

```
提交代码 → L1自动扫描 → 通过?
                         ├─ 是 → 分配审查者
                         └─ 否 → 退回修复
                                     ↓
                              审查者 L2/L3 审查
                                     ↓
                              提交审查报告
                                     ↓
                          通过? ─ 是 → 合并
                           └─ 否 → 修复后重新提交
```

## 四、审查报告模板

```markdown
## 审查报告 — [模块名] @ [日期]

| 检查项 | 状态 | 备注 |
|--------|------|------|
| 安全检查 | ✅/❌ | 具体问题 |
| 正确性 | ✅/❌ | 具体问题 |
| 可靠性 | ✅/❌ | 具体问题 |
| 性能 | ✅/❌ | 具体问题 |
| 可维护性 | ✅/❌ | 具体问题 |

### 发现的问题
- [P0] 问题描述 (文件:行号)
- [P1] 问题描述
- [P2] 问题描述

### 结论: PASS / CONDITIONAL / FAIL
```

## 五、自动化 L1 扫描 (CI)

```bash
# 语法检查
python3 -m py_compile *.py

# 安全扫描
grep -rn "shell=True" *.py
grep -rn "secret_key\s*=\s*['\"]" *.py
grep -rn "password\s*==\s*" *.py

# 导入检查
python3 -c "import app" 2>&1
```

## 六、审查频率

| 场景 | 频率 |
|------|------|
| 日常提交 | 每次 PR L1+L2 |
| 每周 | L2 全面扫描 |
| 每月 | L3 深度审查 |
| 上线前 | L4 全检 |
